Explicación de ciertas funciones

COMMIT
Es la función en el que se crean copias de diferentes versiones de un archivo con cada edición de manera local.
Tenemos acceso a todas estas versiones desde nuestro entorno de desarollo, los commit no son agregados directamente 
a la rama principal, se hace de manera explícita.

PUSH
Una vez el commit está listo con las ediciones deseadas y listo para la integración con la rama principal se 
realiza push. Envia la copia local al repositorio definitivo de github.
